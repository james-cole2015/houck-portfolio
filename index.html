Patch Gap Analyzer — Manual Security Assessment Tool
Personal Project  ·  View on GitHub

A local web application for security assessors performing manual compliance and vulnerability assessments in air-gapped or semi-connected environments. Identifies unpatched CVEs and configuration control failures across Windows and Linux assets by comparing observed system state against authoritative advisory and benchmark data — with no cloud dependency required on-site.

Assessment Pipelines
  • Windows Patch Assessment — ingests Microsoft MSRC CVRF data pre-engagement; on-site, the assessor enters a host's Update Build Revision (UBR) and the tool identifies every missing cumulative update KB and its associated CVEs, including supersedence tracking. Supports 14 Windows targets from Server 2008 R2 through Server 2025
  • Linux Patch Assessment — pulls advisory data from Debian Security Tracker, Ubuntu Canonical OVAL, and Red Hat OVAL v2; generates a collection script requiring no admin rights or internet access on the target; normalizes all three upstream formats into a unified schema for analysis across 20+ distro versions
  • CIS Benchmark Configuration Assessment — parses CIS benchmark XLSX files, generates a pre-populated assessment checklist filtered by profile and Implementation Group, and processes the completed checklist into structured findings
Technical Highlights
  • Implemented from-scratch dpkg and RPM version comparison algorithms per their respective specs, enabling accurate patch gap analysis without requiring native package bindings on the assessment machine
  • Exports findings as NIST OSCAL 1.1.2 POA&M documents (JSON, XML, CSV, HTML) with CVSS-derived completion dates computed from a configurable risk policy table — aligned to federal compliance reporting requirements
  • Enriches advisory data with NVD CVSS scores via the NIST NVD API, normalizing severity labels from distros that don't publish numeric scores
  • Uses Server-Sent Events (SSE) to stream live progress during long-running advisory pulls (2–10+ minutes), avoiding HTTP timeouts without polling
  • Implements a custom server-side session store keyed by UUID cookie — bypassing Flask's cookie-size limit and ensuring no assessed asset data is ever written to disk; server auto-wipes all session data after 2 hours of inactivity
  • Two-phase offline workflow: internet-required pre-engagement data pull, then fully air-gapped on-site execution — the entire project folder is self-contained and portable
